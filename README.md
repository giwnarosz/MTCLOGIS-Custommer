# คู่มือการติดตั้งและใช้งานระบบบริหารจัดการคลังสินค้าและคู่ค้า (Inventory & Supplier Management System)

ระบบนี้พัฒนาขึ้นด้วยสถาปัตยกรรม **Serverless Frontend** (HTML/JS/Tailwind CSS) และเชื่อมต่อฐานข้อมูล **Google Sheets** ผ่าน **Google Apps Script (GAS)** โดยระบบมีประวัติการบันทึกงาน (Audit Log) และสิทธิ์การเข้าถึงแบบ Triple-User Tracking

---

## ขั้นตอนที่ 1: การเตรียมโครงสร้าง Google Sheets

1. เข้าไปที่ [Google Sheets](https://sheets.google.com/) และสร้างสเปรดชีตใหม่ 1 ไฟล์
2. เปลี่ยนชื่อสเปรดชีตเป็นชื่อที่คุณต้องการ (เช่น `ฐานข้อมูลคลังสินค้า`)
3. ทำการเพิ่มชีต (Sheet Tab) ด้านล่างทั้งหมด **3 ชีต** และตั้งชื่อพร้อมสร้างคอลัมน์หัวตาราง (แถวแรก) ดังนี้:

### ชีตที่ 1: `Products`
สร้างหัวตารางในแถวที่ 1 ตั้งแต่คอลัมน์ A ถึง G:
- **A1:** `SKU`
- **B1:** `Name`
- **C1:** `Category`
- **D1:** `UOM`
- **E1:** `Description`
- **F1:** `Quantity`
- **G1:** `DateAdded`

### ชีตที่ 2: `Suppliers`
สร้างหัวตารางในแถวที่ 1 ตั้งแต่คอลัมน์ A ถึง J:
- **A1:** `SupplierID`
- **B1:** `CompanyName`
- **C1:** `TaxID`
- **D1:** `BusinessType`
- **E1:** `ContactPerson`
- **F1:** `Phone`
- **G1:** `Email`
- **H1:** `RegisteredAddress`
- **I1:** `MailingAddress`
- **J1:** `DateAdded`

### ชีตที่ 3: `AuditLogs`
สร้างหัวตารางในแถวที่ 1 ตั้งแต่คอลัมน์ A ถึง I:
- **A1:** `Timestamp`
- **B1:** `ActionType`
- **C1:** `RecordType`
- **D1:** `PrimaryKey`
- **E1:** `Details`
- **F1:** `Creator`
- **G1:** `Editor`
- **H1:** `Preparer`
- **I1:** `Reason`

## ขั้นตอนที่ 2: การติดตั้ง Google Apps Script

1. ที่เมนูด้านบนของ Google Sheets ให้เลือก **ส่วนขยาย (Extensions)** -> **Apps Script**
2. ระบบจะเปิดหน้าต่างแก้ไขโค้ดขึ้นมา ให้ลบโค้ดเดิมทั้งหมดในไฟล์ `รหัส.gs` (หรือ `Code.gs`) ออก
3. เปิดไฟล์ [Code.gs](file:///c:/Users/giwma/OneDrive/%E0%B9%80%E0%B8%94%E0%B8%AA%E0%B8%81%E0%B9%8C%E0%B8%97%E0%B9%87%E0%B8%AD%E0%B8%9B/%E0%B9%80%E0%B8%A7%E0%B9%87%E0%B8%9A%E0%B8%90%E0%B8%B2%E0%B8%99%E0%B8%82%E0%B9%89%E0%B8%AD%E0%B8%A1%E0%B8%B9%E0%B8%A5%202/backend/Code.gs) ในโฟลเดอร์โปรเจกต์นี้ คัดลอกโค้ดทั้งหมดแล้วนำไปวางใน Apps Script
4. กดปุ่ม **บันทึก (Save)** (ไอคอนแผ่นดิสก์) ด้านบน

## ขั้นตอนที่ 3: การ Deploy เพื่อรับ Web App URL

1. กดปุ่ม **การทำให้ใช้งานได้ (Deploy)** ที่มุมขวาบน -> เลือก **การทำให้ใช้งานได้ใหม่ (New deployment)**
2. คลิกไอคอนรูปเฟือง (ข้างคำว่า Select type) -> เลือก **เว็บแอป (Web app)**
3. ตั้งค่าการดีพลอยดังนี้:
   - **คำอธิบาย (Description):** `v1.0` (หรือปล่อยว่างไว้)
   - **เรียกใช้ในฐานะ (Execute as):** `ฉัน (อีเมลของคุณ)`
   - **ผู้มีสิทธิ์เข้าถึง (Who has access):** `ทุกคน (Anyone)` *(สำคัญมาก: หากไม่เลือกข้อนี้ เว็บไซต์จะไม่สามารถดึงข้อมูลได้)*
4. กดปุ่ม **การทำให้ใช้งานได้ (Deploy)**
5. สำหรับการดีพลอยครั้งแรก ระบบจะแสดงปุ่ม **ให้สิทธิ์เข้าถึง (Authorize access)** ให้กดคลิกเพื่ออนุญาตสิทธิ์เข้าถึงสเปรดชีตของคุณ
   - *หมายเหตุ: หากแสดงหน้าต่าง Warning ให้กด **Advanced** -> กด **Go to ... (unsafe)** เพื่อยืนยันให้สิทธิ์เข้าถึง*
6. หลังจากดีพลอยเสร็จสิ้น ระบบจะแสดง **URL ของเว็บแอป (Web app URL)** ให้คลิกปุ่ม **คัดลอก (Copy)** เก็บไว้

## ขั้นตอนที่ 4: การเชื่อมต่อระบบและสิทธิ์การบันทึกข้อมูล (Frontend)

1. เปิดไฟล์ [app.js](file:///c:/Users/giwma/OneDrive/เดสก์ท็อป/เว็บฐานข้อมูล 2/js/app.js) ด้วยโปรแกรมแก้ไขโค้ด (เช่น VS Code, Notepad++)
2. นำ **Web App URL** ที่คัดลอกในขั้นตอนที่ 3 ไปกำหนดที่ตัวแปร `API_URL` ที่บรรทัดแรกสุดของไฟล์ ตัวอย่างเช่น:
   ```javascript
   const API_URL = "https://script.google.com/macros/s/AKfycb.../exec";
   ```
3. บันทึกไฟล์ `js/app.js`
4. ดับเบิ้ลคลิกเปิดไฟล์ [index.html](file:///c:/Users/giwma/OneDrive/เดสก์ท็อป/เว็บฐานข้อมูล 2/index.html) ในเครื่องของคุณเพื่อรันผ่านเว็บเบราว์เซอร์
